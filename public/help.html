Cháchara Documentation

The application text input accepts the following commands

/help

Shows this document

/settings

Prints all configured settings for the user. Example:

      useNotifications	true
      jid	        jd@jabber.com
      nick	        john.doe
      mentions[0]	john
      mentions[1]	node
      mentionSounds	enabled
      messageSounds     disabled

ctrl + <
ctrl + >

Not actually a input text command but a keystroke. It changes the active chat pane. Each chat pane can contain a chatroom window or a private message window

/join room

Joins a room. For example, `/join chat` joins chat@conference.jabber.com

/leave

Leave the current active private chat or room.

/signout

Disconnects from the XMPP server.

#room message

Sends a message to a room (chat pane) that is not currently active. In this case the room string only needs to be what comes before the @ character, i.e. for chat@conference.jabber.com the room name would be just chat. Example:

      #staff hello
      #hr you're all fired

/status type message

Sets a new xmpp status for the user. Three status types are supported: chat (available), xa (extended away) and dnd (do not disturb). The third message parameter is optional. Example:

      /status chat office
      /status xa
      /status dnd call

/audio (mentions | messages | all) (on | off)

Changes the audio settings for the user. These settings will persist in the browser's localStorage database. Audio for mentions uses a different sound clip to regular messages. Example:

      /audio messages off
      /audio all on

@nick message

Sends a private message to a user. nick is the string before the '@' in the recipient's jid, i.e. for user@jabber.com nick would be user. Example:

      @john hello
      @joan lunch time!

/mentions add word

Adds a custom mention notification for the user. By default, the user will only get notified when his name is mentioned. This action allows the user to add more words to trigger a mention notification. Example:

      /mentions add lunch # the user gets notified every time the word 'lunch' appears
